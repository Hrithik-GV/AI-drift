01 — BRIEFING

WHAT IS AI-DRIFT

AI-Drift is an intense, high-octane 6-hour hackathon designed to push your engineering limits to the redline. Organized by the Department of Artificial Intelligence & Machine Learning (AIML), this rapid prototyping marathon challenges you to architect, code, and deploy an impactful solution under extreme time constraints.

VENUE 29 May 2026 AIML Lab, 4th Floor, SIT Mangaluru
TIMINGS 09:30 AM – 03:30 PM 6 Hours Active Coding
TEAM SIZE 2 – 4 Members Per Registered Team
CORE FOCUS DOMAIN AI & Cybersecurity FinTech Protection
CHALLENGE

Fraud Shield for First-Time Digital Users — Develop an AI-powered real-time fraud prevention system that protects inexperienced users while they perform digital financial activities.

02 — PROTOCOL

RULES & REGULATIONS

  • 01 SQUAD LIMITS

    2–4 members per team.

  • 02 TIMELINE DISCIPLINE

    Build must happen entirely within the official hackathon duration.

  • 03 STACK FREEDOM

    Complete freedom to use any programming language, framework, or technology stack.

  • 04 CODE ETHICS

    Pre-built templates/UI kits are allowed to optimize time, but all core application logic must be developed live. Copying existing GitHub repositories or submitting historical codebases is strictly banned.

  • 05 VERSION CONTROL

    Teams must maintain a public GitHub repository with a clear README file detailing the tech stack and configuration instructions, submitting the link at 03:30 PM sharp.

  • 06 PROJECT SAFEGUARDS

    Applications must strictly focus on protection, risk analysis, and user awareness — no offensive exploits or harmful cybersecurity activities allowed. Use demo or simulated transactions for validation testing.

  • 07 DELIVERABLES

    A functional MVP or working prototype must be demonstrated live during judging evaluations.
